# LBKI Backlog Planner

> 🗂️ Инструмент для ИТ-менеджеров: управление бэклогом, ресурсное планирование и автоматическое построение Waterfall-дорожных карт.

## 📖 Описание
Клиентское веб-приложение для структурирования проектов по иерархии **Проект → Эпик → Фича → Задача**, назначения ресурсов с учётом ёмкости и множителей, а также автоматического расчёта реалистичных сроков. Поддерживает зависимости задач, учёт рабочих дней (Пн–Пт), подсветку срывов сроков и визуализацию через диаграмму Ганта (Mermaid).

Разработано на чистом `HTML + CSS + JS` с соблюдением принципов **функционального программирования**, иммутабельного состояния и подробного логирования для отладки.

## 🚀 Запуск и использование
1. **Локальный запуск**: Откройте `index.html` в современном браузере (Chrome, Firefox, Edge, Safari).
2. **Настройка проекта**: Нажмите `⚙️ Настройки проекта` → укажите диапазон дат.
3. **Создание данных**: Через кнопки `📋 Эпики`, `📋 Фичи`, `📋 Ресурсы` добавьте справочники.
4. **Добавление задач**: Нажмите `+ Задача`, заполните поля, укажите пререквизиты (ID через запятую).
5. **Расчёт плана**: Нажмите `📅 Сформировать план`. Даты рассчитаются автоматически.
6. **Визуализация**: Нажмите `📊 Диаграмма Ганта` для просмотра roadmap.
7. **Сохранение**: Данные сохраняются в браузере автоматически. Для переноса используйте `💾 Сохранить JSON` / `📂 Загрузить JSON`.

## 📊 Логика планирования (Waterfall)
Алгоритм работает в 4 этапа:
1. **Топологическая сортировка** (Kahn's algorithm) → гарантирует выполнение пререквизитов до зависимых задач.
2. **Composite Priority** → `(Эпик × 100) + (Фича × 10) + Задача`. Чем ниже число, тем выше приоритет.
3. **Resource Calendar** → для каждого дня недели ведётся учёт занятости ресурса. Поиск слота идёт пошагово, пропуская выходные.
4. **Deadline Check** → если `actualEnd > finishNoLaterThan`, задача помечается `isViolated: true`, строка таблицы краснеет, статус меняется на `risk`.

## 💾 Сохранение данных и восстановление
- **Автосохранение**: Любое изменение состояния (`updateState`) мгновенно пишется в `localStorage`.
- **Восстановление сессии**: При открытии страницы, если найдены сохранённые данные, появляется `confirm()` с предложением продолжить работу.
- **Защита от закрытия**: `beforeunload` активирует нативное предупреждение браузера. Данные при этом уже сохранены.
- **Ручной бэкап**: JSON-файл содержит полную копию `state`, включая `nextId`, календари и логи (если включены).

## 🐛 Отладка и логи
Все критические шаги логируются в `console.log` с префиксами:
| Префикс | Описание |
|---------|----------|
| `[INIT]` | Инициализация, восстановление сессии |
| `[STATE]` | Изменения состояния, CRUD-операции |
| `[PLAN]` | Расчёт дат, сдвиги из-за выходных, срывы сроков |
| `[RENDER]` | Перерисовка таблицы, применение фильтров |
| `[GANTT]` | Генерация Mermaid-синтаксиса, ошибки рендера |
| `[STORAGE]` | Запись/чтение `localStorage` |

Откройте **DevTools (F12) → Console** для трассировки алгоритма или поиска причин расчёта дат.

## ⚠️ Известные ограничения
- Приложение работает полностью на клиенте. Нет бэкенда и синхронизации между устройствами (используйте JSON-экспорт).
- Mermaid.js подгружается с CDN. Для оффлайн-работы скачайте `mermaid.min.js` и измените путь в `index.html`.
- При более чем ~5000 задач возможна деградация производительности в браузере (рекомендуется вынести `findAvailableSlot` в Web Worker).
